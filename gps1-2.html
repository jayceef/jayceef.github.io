Pear Programming is better than Orange Programming

Pair programming is an attempt by two people to work together on the same code, whether locally or remotely.
Dev Boot Camp likes pair programming for many reasons. Pair programming is more efficient than writing code by yourself, because it gives the pair opportunities to collaborate on ideas and check each others' work. For example, pair programming combines the knowledge of both people, therefore expanding the possibilities of writing decent or correct code, as well as better design in general. There's also the learning- teaching paradigm, whereby one partner who might know more about one thing and help out, then vice versa.
  • The driver does what the navigator instructs him to do.
  • The driver is responsible for writing the code or text as best he/she knows how.
  • The driver is responsible for communicating with the navigator efficiently.
  • The navigator tells the driver what to do.
  • The navigator is responsible for keeping track of time.
  • The navigator checks in with the driver to make sure all are on the same page (excuse the pun)
